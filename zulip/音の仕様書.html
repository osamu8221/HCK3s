「かえるの合唱」自動合奏システムの段階的実装とテストガイド

Step 1 楽曲データの定義と数値化（Arduino側）

Arduinoが演奏データを扱えるよう、「かえるの合唱」の楽譜を周波数（Hz）の配列に変換します。休符は 0.0 として扱います。

  • 対象曲である「かえるの合唱」のド〜ラまでの周波数を定義します。
  • MusicData.h に楽曲の配列を作成します。
// MusicData.h の実装例
// 周波数定義：ド=261.6, レ=293.7, ミ=329.6, ファ=349.2, ソ=392.0, ラ=440.0, 休符=0.0

// かえるのうた（最初の2小節）
// 1行を4拍（1小節）に分けて書くと管理しやすいです。
float melody[] = {
    // ド    レ    ミ    ファ
    261.6, 293.7, 329.6, 349.2, 
    // ミ    レ    ド    休符
    329.6, 293.7, 261.6,   0.0, 
    // ミ    ファ  ソ    ラ
    329.6, 349.2, 392.0, 440.0, 
    // ソ    ファ  ミ    休符
    392.0, 349.2, 329.6,   0.0
};

// 実際の配列サイズを計算しておく
int melodyLength = sizeof(melody) / sizeof(melody[0]);

✅ 成功の目安

  • コードをコンパイルした際、エラーが出ないこと。
  • 休符（0.0）を含めたメロディの配列データが出力できる状態になっていること。

Step 2 Processingでの音源基盤と音色の作成（Processing単体）

シリアル通信でArduinoとつなぐ前に、PC側（Processing）単体で狙った音が鳴るか、テスト配列を用いて動作確認を行います。

  • AudioPlayer.pde に発振器を用いた音声波形の生成処理を実装します。
  • Processingのコード内にテスト用の配列を用意し、音が正しく出るか確認します。
// Processingテスト用配列
float[] testMelody = {
    261.6, 293.7, 329.6, 349.2, 392.0, 440.0
};

✅ 成功の目安

  • Processing上でプログラムを実行した際、testMelody の配列に従って「ド・レ・ミ・ファ・ソ・ラ」とエラーなく音が鳴ること。
  • 音色の判別：ギロ、バイオリン、トランペット、オルガンなどの音色を作り分けた時、聞いて明らかに違う音色になっていること。ノイズや音割れが発生していないこと。

Step 3 Arduinoからの周波数送信テスト（Arduino単体）

Processingと繋ぐ前に、Arduino単体でシリアルプリントが正しいタイミングで出ているか確認します。

  • InstrumentController から、計算されたタイミングで周波数情報がシリアル通信に乗るかテストします。

✅ 成功の目安

  • Arduino IDEの「シリアルモニタ」を開いた時、一定のリズム（例：500ms間隔）で 261.6 などの数値が改行されながら連続して表示されること。
  • 休符のタイミングでは 0.0 が出力されていること。余計な文字化けが含まれていないこと。

Step 4 ArduinoとProcessingの連携（シリアル通信テスト）

Arduinoから送られてきた周波数（Hz）データをProcessingの SerialReceiver.pde で受け取ります。

✅ 成功の目安

  • Arduinoを繋いでProcessingを実行すると、自動的に「かえるの合唱」のメロディがスピーカーから流れること。
  • Processingのコンソール画面に Received: 261.6 Hz と受信ログが正しく表示されており、通信の遅延（もたつき）を感じないこと。

Step 5 テンポ変更機能の実装

指揮者の動き（BPM）に合わせてテンポを変えるための処理 updateTempo() を組み込みます。

✅ 成功の目安

  • Arduino側のテスト用変数 test_BPM = 60; を test_BPM = 120; に書き換えて再実行した際、Processingから流れる音楽のスピードが正確に2倍の速さになること。
  • テンポを速くしても、音が重なったり途切れたりせず、メロディが崩れないこと。

Step 6 視覚演出の同期確認（LEDマトリクス編）

Arduino UNO R4 WiFi に搭載されているLEDマトリクスを用いて、演奏状態の表示が音と同期しているか確認します。

  • drawFrogMatrix() 関数を使用し、演奏状態に応じてLEDマトリクスの表示を更新します。

✅ 成功の目安

  • 音が鳴っている間（休符以外）はLEDが点灯（または特定のパターンを表示）し、休符の時は消灯するなど、音と光のタイミングが完全に一致していること。
  • LED表示の処理を追加したことで、音楽のテンポに遅延が発生していないこと。

Step 7 視覚演出の同期確認（サーボモータ編）

マイクロサーボモータ（SG92R）を用いて、指定された音階に対応する「紙のカエル」が起き上がる物理演出を制御します。

  • moveFrog() 関数を使用し、音階に応じたサーボモータを駆動させます。

✅ 成功の目安

  • ド（261.6Hz）が鳴った瞬間に、1番のサーボモータ（ドのカエル）がピョコッと動き、音が消えるタイミングで元に戻ること。
  • サーボモータが動く際の処理負荷（電圧降下など）によって、Arduinoのシリアル送信（音楽のテンポ）が遅延したり、システムが再起動したりしないこと。

Step 8 同期制御ユニットとの全体連携（結合テスト）

センサ制御部・同期制御部のArduinoと結合し、システム全体を通したテストを行います。

✅ 成功の目安

  • 演奏開始： 同期制御ArduinoからSTART信号を受け取るまで楽器側は沈黙し、受け取った瞬間に1音目（ド）が鳴り始めること。
  • テンポ更新： 演奏の途中で新しいBPM値が送られてきた場合、次の音符から即座に新しいテンポに切り替わること。
  • 輪唱成立： 複数の楽器制御ユニットを繋いだ時、それぞれの担当パートが設計通りのズレで演奏を開始し、全体として音楽的に破綻していないこと。
