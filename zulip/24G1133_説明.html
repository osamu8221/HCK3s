🎯 プロジェクトの概要

プロジェクト題目

「加速度センサを用いた身体動作による輪唱制御システムの構築と視覚的フィードバックの効果」

プロジェクトの目的

主要目的： 指揮者の身体動作を入力として利用し、音や光、映像表現を連動させることで、視覚的・聴覚的に楽しめる体験型演出システムを開発することである。

本プロジェクトの核となる技術的課題は、複数のデバイスを連動させる際に、センサ情報の取得、演奏開始の判定、テンポ変化の伝達、各デバイスへの同期指示を適切に行うことです。これらの処理が不安定であると、演奏開始タイミングのずれやテンポ変化の反映遅れが発生し、演出の一体感が損なわれます。

技術的背景

近年のデジタル技術発展により、利用者自身が演出に参加できるコンテンツへの関心が高まっています。従来の鑑賞型コンテンツでは利用者は受動的でしたが、利用者の動作に応じて演出が変化するシステムでは、参加感や没入感が大幅に向上します。

最終的なゴール

指揮者の動きに応じて複数のデバイスが同期して動作し、音や光、映像が一体となって変化する体験型演出システムを完成させる」

✅ 検証と妥当性確認 (V&V)

有効指標 (MOE: Measure of Effectiveness)

ユーザ視点での評価指標
  • 📌 直感的操作性と同期感：遅延や違和感なく同期
  • 📌 音楽的成立：複数楽器による安定した輪唱表現
  • 📌 没入感：参加感を備えた体験型演出

性能指標 (MOP: Measure of Performance)

技術的に確認可能な指標
  • ⚡ エンドツーエンド遅延
  • ⚡ デバイス間の同期ジッタ
  • ⚡ 動作認識の処理時間
  • ⚡ アクチュエータの応答時間

技術性能指標 (TPM: Technical Performance Measure)

開発中に継続監視する指標

1. IMU データ取得周期

サンプリング間隔のばらつきが BPM 算出誤差につながらないか監視

2. ビート検出性能

誤検出率と見逃し率を記録・評価

3. 通信状態

応答時間、通信失敗回数、テンポ更新反映時間を監視

4. 処理負荷

演奏処理と視覚演出を同時実行時の遅延・フレーム落ち監視

🧪 テスト戦略

システムテスト

指揮動作の入力から音声出力および視覚演出開始までの
システム全体の応答性能を評価

目標：10ms 以下のエンドツーエンド遅延

音楽インタラクション研究での標準目標値に準拠

結合テスト

3つの制御部（センサ制御、同期制御、楽器制御）が
期待通りに情報を受け渡すかを評価

監視項目
  • START信号配信成功
  • テンポ同期成功
  • 発音成功
  • 視覚演出同期成功

単体テスト

個別モジュールの妥当性を確認

1. IMU 取得部

加速度・角速度の取得値とノイズ除去後の安定性

2. テンポ算出部

BPM 算出値の安定性と入力テンポとの一致度

3. サーボ駆動部

信号受信から目標角度到達までの応答時間（100ms 以下が目標）

⚠️ リスク管理

主要技術的リスク

  • IMU センサによる誤検出 → 加速度データの平滑化処理で対策
  • Arduino 間通信の同期ずれ → 同期制御部による状態管理で対策
  • サーボモータ制御による処理遅延 → 段階的な統合テストで早期発見

リスク低減策

  • ✓ 加速度データの平滑化処理の実装
  • ✓ 同期制御部による状態管理とポーリング方式の採用
  • ✓ 各制御部の独立開発を可能にする設計
  • ✓ 一部機能の問題が他機能開発に影響しない構造
